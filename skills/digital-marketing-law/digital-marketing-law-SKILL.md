# デジタルマーケティング法令参照スキル

## 概要

このスキルは、e-Gov法令検索MCPサーバー（law-mcp-server）を使用して、デジタルマーケティング業務に関連する法令を効率的に検索・参照するためのガイドです。

### 対象ツール

- `search_laws` - キーワードで法令を検索し、canonical LawID を取得
- `fetch_law` - LawID で法令本文を取得（オプションで revisionDate 指定可）
- `check_consistency` - 文書と法令の整合性をチェック（lawIds 必須）
- `summarize_law` - 法令の要約を生成（オプションで条文番号を絞り込み可）

### APIの制約

- Rate limit: e-Gov API の制限に従う
- Cache TTL: デフォルト900秒（15分）
- タイムアウト: 15秒（設定可能）

## 法令カテゴリと検索パターン

### 1. 表示規制（うそ・誇大・ステマ等）

#### 不当景品類及び不当表示防止法（景品表示法）

**正式名称**: 不当景品類及び不当表示防止法
**略称**: 景品表示法、景表法

**主な規制内容**:

- 優良誤認表示（第5条第1号）
- 有利誤認表示（第5条第2号）
- ステルスマーケティング（第5条第3号、告示）
- No.1表示のエビデンス要件
- 口コミ・体験談の適正表示

**検索キーワード**:

```
基本検索: search_laws keyword="不当景品類及び不当表示防止法"
略称検索: search_laws keyword="景品表示法"
条文確認: fetch_law で取得後、第5条を参照
```

**実務での活用例**:

- 広告クリエイティブレビュー時の第5条確認
- ステマ規制の告示内容と対象範囲の確認
- 比較広告作成時の根拠要件の確認
- インフルエンサーマーケティングガイドライン策定

**よく参照する条文・告示**:

- 第5条（不当な表示の禁止）
- 景品類等の指定の告示（昭和37年公正取引委員会告示第3号）
- 一般消費者が事業者の表示であることを判別することが困難である表示の運用基準

#### 特定商取引に関する法律（特商法）

**正式名称**: 特定商取引に関する法律
**略称**: 特商法、特定商取引法

**主な規制内容**:

- 通信販売の広告表示義務（第11条）
- 誇大広告の禁止（第12条）
- 電子メール広告の規制（第12条の3、第12条の4）
- 返品特約の表示義務
- LP・EC広告の必須記載事項

**検索キーワード**:

```
基本検索: search_laws keyword="特定商取引に関する法律"
略称検索: search_laws keyword="特定商取引法"
条文確認: fetch_law で取得後、第11条・第12条を参照
```

**実務での活用例**:

- EC/LP制作時の必須記載事項チェック
- 特定商取引法に基づく表記ページの作成
- メールマーケティングの適法性確認
- アフィリエイト広告の表示要件確認

**よく参照する条文**:

- 第11条（通信販売における誇大広告等の禁止）
- 第12条（誇大広告等の禁止）
- 第12条の3（電子メール広告の表示義務）
- 第12条の4（再勧誘等の禁止）

#### 消費者契約法

**正式名称**: 消費者契約法

**主な規制内容**:

- 不実告知・断定的判断の提供による取消し（第4条）
- 不利益事実の不告知による取消し

## 実践ワークフロー

### ワークフロー1: プライバシーポリシー作成

**目的**: 個人情報保護法、電気通信事業法、Cookie規制に準拠したプライバシーポリシーの作成

**手順**:

1. **基本法令の検索とLawID取得**

```
search_laws: keyword="個人情報の保護に関する法律"
→ 返された canonical LawID（例: H15HO57）を控える
```

2. **法令本文の取得と関連条文の確認**

```
fetch_law: lawId=<取得したLawID>
→ 第21条（利用目的の特定）
→ 第27条（第三者提供の制限）
→ 第31条（個人関連情報）を重点確認
```

3. **外部送信規律の確認**

```
search_laws: keyword="電気通信事業法"
fetch_law: lawId=<取得したLawID>
→ 第27条の12を確認
```

4. **文書の整合性チェック**

```
check_consistency:
  documentText: <作成したプライバシーポリシー本文>
  lawIds: [<個人情報保護法のLawID>, <電気通信事業法のLawID>]
```

### ワークフロー2: 広告クリエイティブの法令チェック

**目的**: 景品表示法、特商法、薬機法（該当する場合）への適合確認

**手順**:

1. **商材に応じた法令の特定と LawID 取得**

```
一般商品: search_laws で景品表示法、特商法の LawID を取得
化粧品: + 薬機法
金融商品: + 金融商品取引法
```

2. **法令の取得と重要条文の確認**

```
fetch_law: lawId=<景品表示法のLawID> → 第5条（不当表示の禁止）
fetch_law: lawId=<特商法のLawID> → 第11条、第12条（誇大広告の禁止）
```

3. **広告文との整合性チェック**

```
check_consistency:
  documentText: <広告文・キャッチコピー>
  lawIds: [<景品表示法のLawID>, <特商法のLawID>]
```

4. **具体的な表現の確認**

- No.1表示 → 景品表示法 + 実証データの有無
- 比較表現 → 景品表示法 + 比較対象の明示
- 体験談 → 景品表示法 + ステマ告示

### ワークフロー3: メールマーケティングの適法性確認

**目的**: 特定電子メール法、個人情報保護法への準拠確認

**手順**:

1. **法令の検索と取得**

```
search_laws: keyword="特定電子メールの送信の適正化等に関する法律"
fetch_law: lawId=<取得したLawID> → 第3条（オプトイン）、第4条（表示義務）を確認
search_laws: keyword="個人情報の保護に関する法律"
fetch_law: lawId=<取得したLawID> → 第27条（第三者提供）を確認
```

2. **配信リスト管理の確認**

```
check_consistency:
  documentText: <配信リスト取得方法の説明文書>
  lawIds: [<特定電子メール法のLawID>, <個人情報保護法のLawID>]
```

3. **メール本文の要件チェック**

- 送信者の氏名・名称
- オプトアウト方法（配信停止）の表示
- 問い合わせ先の記載

### ワークフロー4: プラットフォーム取引の公正性評価

**目的**: 独占禁止法、デジタルプラットフォーム取引透明化法への適合確認

**手順**:

1. **関連法令の検索と取得**

```
search_laws: keyword="私的独占の禁止及び公正取引の確保に関する法律"
fetch_law: lawId=<取得したLawID> → 第19条（不公正な取引方法）を確認
search_laws: keyword="特定デジタルプラットフォームの透明性及び公正性の向上に関する法律"
fetch_law: lawId=<取得したLawID>
```

2. **取引条件の整合性チェック**

```
check_consistency:
  documentText: <プラットフォーム利用規約・契約書>
  lawIds: [<独占禁止法のLawID>, <デジタルプラットフォーム透明化法のLawID>]
```

3. **具体的なチェックポイント**

- 取引条件の一方的変更の有無
- 自己優遇の禁止
- アルゴリズムの透明性
- データポータビリティ

### ワークフロー5: 改正法の影響調査

**目的**: 法改正による既存施策への影響範囲の把握

**手順**:

1. **最新版の法令を取得**

```
search_laws: keyword=<対象法令の正式名称>
fetch_law: lawId=<取得したLawID>
→ 最新の法令本文を確認
```

2. **新旧条文の比較（revisionDate を活用）**

```
fetch_law: lawId=<LawID>, revisionDate=<旧版の日付>
fetch_law: lawId=<LawID>（最新版）
→ 両者を比較して変更箇所を特定
```

3. **既存文書の再チェック**

```
check_consistency:
  documentText: <既存のプライバシーポリシー等>
  lawIds: [<改正法のLawID>]
```

## ベストプラクティス

### 検索のコツ

#### 正式名称と略称の使い分け

- 初回検索: 正式名称で確実にヒット
- 法令ID取得後: 略称でも可（効率化）

#### 複数法令の横断検索

関連法令をまとめて確認する場合:

```
1. 個別に法令検索・取得
2. 各法令の関連条文を特定
3. check_consistencyで一括チェック
```

#### 改正前後のバージョン管理

- `fetch_law` の `revisionDate` パラメータで特定時点の法令を取得
- 施行日を確認して適切なバージョンを参照
- 経過措置の有無を確認

### 整合性チェックの効果的な使い方

#### 結果の解釈と対応

**aligned（適合）**:

- 問題なし、そのまま進行可能

**potential_mismatch（潜在的不一致）**:

- 表現の見直しを検討
- 法令専門家への確認推奨

**not_found（参照なし）**:

- 該当条文の引用漏れの可能性
- 必要に応じて追記

### エラーハンドリング

#### APIエラーへの対応

- 429（Rate Limit）: 待機後リトライ
- 404（Not Found）: LawIDの再確認
- 503（Service Unavailable）: バックオフ後リトライ

#### 検索結果が0件の場合

- キーワードを見直す（略称 ↔ 正式名称）
- より一般的な用語で再検索
- 関連法令から辿る

## トラブルシューティング

### 法令が見つからない場合

**原因**:

- 法令名の表記ゆれ（正式名称・略称・旧称）
- 既に廃止された法令の検索
- 政令・省令と法律の混同

**対処**:

1. 正式名称で再検索
2. e-Gov法令検索サイトで手動確認
3. 関連法令から辿る

### 検索結果が多すぎる場合

**対処**:

1. より具体的なキーワードを追加
2. 条文番号を指定
3. 法令の種類（法律・政令・省令）を限定

### 特定時点の法令が取得できない場合

**原因**:

- 指定した `revisionDate` に対応するバージョンが存在しない

**対処**:

1. e-Gov法令検索サイトで改正履歴を手動確認
2. 官報を参照
3. 法令データベース（第一法規、LEX/DBなど）を活用

## 注意事項

### 法的助言ではない

このスキルはあくまで法令の検索・参照を支援するものであり、法的助言を提供するものではありません。具体的な法的判断が必要な場合は、弁護士等の専門家にご相談ください。

### 最新性の確認

法令は随時改正されます。重要な判断の前には:

1. `fetch_law` で最新版の法令本文を取得して確認
2. 施行日・経過措置を確認
3. 必要に応じて官報や専門家の見解も確認

### ガイドライン・指針の参照

法令本文だけでなく、以下も重要:

- ガイドライン（経済産業省、消費者庁等）
- Q&A（個人情報保護委員会等）
- パブリックコメント
- 業界自主規制

これらは法令検索APIでは取得できないため、別途確認が必要です。

## まとめ

このスキルを活用することで、デジタルマーケティング業務で必要な法令を効率的に検索・参照でき、コンプライアンスリスクを低減できます。

**重要なポイント**:

1. 商材・施策に応じて関連法令を特定
2. 正式名称と略称を使い分けて効率的に検索
3. `check_consistency`で文書の法令適合性を確認
4. `fetch_law` の `revisionDate` を活用し、常に適切なバージョンを参照
5. 疑問点は法律専門家に相談

**このスキルで実現できること**:

- プライバシーポリシーの法令適合性確認
- 広告クリエイティブの表示規制チェック
- メールマーケティングの適法性確認
- プラットフォーム取引の公正性評価
- 改正法の影響範囲の迅速な把握

法令遵守の確認作業を自動化・効率化し、より戦略的な業務に時間を使えるようにしましょう。
